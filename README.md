## Завдання 1. Копіювання файлів за розширенням

Скрипт рекурсивно обходить вихідну директорію, копіює файли в директорію призначення та розкладає їх по піддиректоріях відповідно до розширення (наприклад `.txt`, `.js`; файли без розширення — у `other`).

### Запуск

```bash
python -m src.copy_script -s /path/to/source -d /path/to/dest
```

| Аргумент | Опис |
|----------|------|
| `-s` / `--source` | Вихідна директорія (обов’язково) |
| `-d` / `--destination` | Директорія призначення (за замовчуванням `./dist`) |
| `-l` / `--limit` | Максимальна глибина обходу (за замовчуванням `5`) |

Код обробляє `ValueError` для невалідних шляхів, а також `PermissionError` і `OSError` під час читання/копіювання.

Реалізація: [`src/file_copy.py`](src/file_copy.py), CLI: [`src/copy_script.py`](src/copy_script.py).

## Завдання 3. Порівняння алгоритмів сортування

Порівняно **insertion sort**, **merge sort**, **quicksort** і вбудований **Timsort** (`sorted`) на чотирьох типах даних:

- випадковий масив;
- відсортований масив;
- обернений (найгірший для insertion);
- майже відсортований.

Розміри вибірок: **10**, **100**, **1000**, **10000**. Бенчмарк: [`src/sorting/benchmark.py`](src/sorting/benchmark.py).

```bash
python -m src.sorting.benchmark
```

### Теоретична складність

| Алгоритм | Середній випадок | Найгірший випадок | Найкращий випадок |
|----------|------------------|-------------------|-------------------|
| Insertion sort | \(O(n^2)\) | \(O(n^2)\) | \(O(n)\) (вже відсортований) |
| Merge sort | \(O(n log n)\) | \(O(n log n)\) | \(O(n log n)\) |
| Quicksort | \(O(n log n)\) | \(O(n^2)\) | \(O(n log n)\) |
| Timsort | \(O(n log n)\) | \(O(n log n)\) | \(O(n)\) (майже/повністю відсортований) |

Timsort поєднує ідеї **merge sort** (стабільне злиття runs) і **binary insertion sort** (швидке сортування коротких майже впорядкованих фрагментів), тому на реальних і майже відсортованих даних він суттєво виграє.

### Результати вимірювань (мс)

Часи отримано одним прогоном `python -m src.sorting.benchmark` (залежать від машини; важливі відносні співвідношення).

### Висновки

1. **Insertion sort** дуже швидкий на невеликих об'ємах даних чи майже відсортованих даних, а далі програє в швидкості в інщих алгоритмах.
2. **Merge sort** стабільно тримає \(O(n \log n)\) на всіх типах даних.
3. **Quicksort** у середньому швидший за merge sort на випадкових даних, але завжди поступається tim sort.
4. **Timsort** найшвидший у всіх сценаріях.
5. Емпірично підтверджено, що Timsort найшвидший серед 4х імплементацій. По цій причині `sorted` / `list.sort` у Python використовують Timsort.
